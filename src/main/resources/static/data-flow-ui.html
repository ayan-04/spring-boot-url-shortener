🔵 Flow Channel 1: URL Shortening

Create a shortened URL from a long URL with optional expiration

POST /api/v1/data/shorten
🌐
HTTP Client
External Request

📥 Request Body (JSON)

{
  "longUrl": "https://example.com/very/long/path",
  "expiresAt": "2026-12-31T23:59:59Z",
  "redirectType": "TEMPORARY"
}

📤 Expected Response

{
  "shortUrl": "00003d3",
  "longUrl": "...",
  "expiresAt": "...",
  "createdAt": "...",
  "redirectType": "TEMPORARY"
}
🎮
Controller Layer
UrlController.java → shortenUrl()

📥 Input

  • @RequestBody ShortenUrlRequest request
  • @Valid triggers validation

⚙️ Processing

  • Extracts longUrl from request
  • Extracts expiresAt from request
  • Delegates to urlService.shortenUrl()
📦
DTO Validation
ShortenUrlRequest.java

📥 JSON → Java Object

ShortenUrlRequest {
  longUrl: String
  expiresAt: OffsetDateTime
  redirectType: RedirectType (TEMPORARY | PERMANENT)
}

✅ Validation Rules

  • @NotBlank - longUrl required
  • @Pattern("^https?://.*") - URL format
⚙️
Service Layer
UrlService.java → shortenUrl()

📥 Input Parameters

longUrl: String
expiresAt: OffsetDateTime

⚙️ Business Logic

  • 1. Check if URL exists in DB
  • 2. If exists → return existing
  • 3. Create new Url entity
  • 4. Save to get auto-generated ID
  • 5. Generate shortUrl via Base62
  • 6. Save again with shortUrl
🔢
Utility Layer
Base62Encoder.java → encodeToLength()

📥 Input

value: 12345 (database ID)
length: 7

📤 Output

shortUrl: "00003d3"

Character Set:
0-9  → '0'-'9' (digits)
10-35 → 'a'-'z' (lowercase)
36-61 → 'A'-'Z' (uppercase)
🗄️
Database Layer
UrlRepository.java + PostgreSQL

📥 Entity Saved

Url {
  id: 12345 (auto)
  shortUrl: "00003d3"
  longUrl: "https://..."
  clickCount: 0
  expiresAt: 2026-12-31
  createdAt: 2026-02-01
  redirectType: TEMPORARY
}

📤 SQL Executed

INSERT INTO urls 
  (long_url, expires_at, click_count, redirect_type)
VALUES ('https://...', '2026-12-31', 0, 'TEMPORARY');

UPDATE urls 
SET short_url = '00003d3' 
WHERE id = 12345;

📤 HTTP Response 200 OK

{
    "shortUrl": "00003d3",
    "longUrl": "https://example.com/very/long/path",
    "expiresAt": "2026-12-31T23:59:59Z",
    "createdAt": "2026-02-01T10:30:00Z",
    "redirectType": "TEMPORARY"
}

🗃️ Database Schema: urls table

Column Type Constraints Description
id BIGINT PRIMARY KEY, AUTO Unique identifier
short_url VARCHAR(7) UNIQUE, INDEX 7-char short code
long_url TEXT NOT NULL Original URL
click_count BIGINT NOT NULL, DEFAULT 0 Redirect counter
expires_at TIMESTAMP NULLABLE Expiration time
created_at TIMESTAMP NOT NULL, AUTO Creation time
redirect_type VARCHAR(20) NOT NULL, DEFAULT 'TEMPORARY' TEMPORARY (302) or PERMANENT (301)

🟢 Flow Channel 2: URL Redirection

Redirect a short URL to its original long URL and track click count

GET /api/v1/{shortUrl}
🌐
Browser/Client
HTTP GET Request

📥 Request

GET /api/v1/00003d3 HTTP/1.1
Host: localhost:8080

📤 Expected Response

HTTP/1.1 301|302 (based on redirectType)
Location: https://example.com/...
🎮
Controller Layer
UrlController.java → redirectToUrl()

📥 Input

  • @PathVariable shortUrl = "00003d3"
  • HttpServletResponse response

⚙️ Processing

  • Call urlService.getUrlByShortUrl()
  • If found → increment click count
  • Check url.getRedirectType()
  • Set 301 (PERMANENT) or 302 (TEMPORARY)
  • Set Location header for redirect
⚙️
Service Layer
UrlService.java

⚙️ getUrlByShortUrl()

  • Query: findByShortUrl()
  • Check: isExpired()
  • Return: Optional<Url>

⚙️ incrementClickCount()

  • @Transactional atomic update
  • SQL: click_count = click_count + 1
🔀
Decision Point
Response Generation

✅ URL Found & Valid

HTTP/1.1 301|302 (configurable)
Location: https://example.com/path

→ PERMANENT (301): Cached by browser
→ TEMPORARY (302): Every click counted
→ Click count incremented

❌ Not Found / Expired

HTTP/1.1 404 Not Found

→ URL doesn't exist in DB
→ Or URL has expired

📤 HTTP Response 301/302 Redirect

HTTP/1.1 301 (PERMANENT) or 302 (TEMPORARY)
Location: https://example.com/very/long/path
Content-Length: 0

📌 Note: TEMPORARY (302) recommended for accurate click counting

🟣 Flow Channel 4: Scheduled Cleanup

Automatic background task to detect and log expired URLs

⏰ @Scheduled(fixedRate = 3600000) — Every 1 hour
⏰
Spring Task Scheduler
UrlShortenerApplication.java

⚙️ Bootstrap Configuration

@SpringBootApplication
@EnableCaching      ← Redis cache
@EnableScheduling   ← Enables @Scheduled

📤 Scheduler Details

  • Trigger: Every 3,600,000 ms
  • Thread: scheduling-1
  • Autonomous execution
🧹
Cleanup Task
ExpiredUrlCleanupTask.java

📥 Trigger

@Scheduled(fixedRate = 3600000)
@Transactional
public void cleanupExpiredUrls()

⚙️ Processing

  • Count expired URLs
  • Log if count > 0
  • (Optional) Delete expired
🗄️
Database Query
UrlRepository.java

📥 JPQL Query

@Query("SELECT COUNT(u) FROM Url u 
  WHERE u.expiresAt IS NOT NULL 
  AND u.expiresAt < CURRENT_TIMESTAMP")

📤 Output → Logger

    "level": "INFO",
    "logger": "c.u.s.ExpiredUrlCleanupTask",

🛡️ GlobalExceptionHandler.java

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Map> handleValidationExceptions(...) {
        // Extract field errors
        // Build error response map
        return ResponseEntity.badRequest().body(response);
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<Map> handleGeneralException(...) {
        // Generic error response
        return ResponseEntity.status(500).body(response);
    }
}
